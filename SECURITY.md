# Security Policy

## Supported Versions

Fastlate is currently in early development. Security fixes are provided for the
latest version available on the `main` branch and for the latest published
GitHub Release.

| Version | Supported |
| ------- | --------- |
| latest  | Yes       |
| older   | No        |

## Reporting a Vulnerability

Please do not report security vulnerabilities through public GitHub issues.

Use GitHub Private Vulnerability Reporting when it is available:

https://github.com/lucas081297/fastlate/security/advisories/new

You can also review the repository security page here:

https://github.com/lucas081297/fastlate/security

When possible, include:

- A clear description of the vulnerability.
- Steps to reproduce the issue.
- The affected version, commit, or configuration.
- Any potential impact you identified.
- Suggested fixes or mitigations, if available.

## Disclosure and Response Timeline

We will try to acknowledge vulnerability reports within 7 days.

After acknowledgement, we will investigate the report, coordinate any required
fix, and publish a security update as soon as practical. For confirmed
vulnerabilities, we aim to provide a fix or mitigation within 90 days whenever
possible.

Public disclosure should wait until a fix or mitigation is available, unless
there is active exploitation or another clear public safety reason to disclose
sooner.

Credit will be given to the reporter when appropriate and requested.

# Politica de Seguranca

## Versoes Suportadas

O Fastlate esta em desenvolvimento inicial. Correcoes de seguranca sao
fornecidas para a versao mais recente disponivel na branch `main` e para a
ultima GitHub Release publicada.

| Versao | Suportada |
| ------ | --------- |
| latest | Sim       |
| antigas | Nao      |

## Reportando uma Vulnerabilidade

Por favor, nao reporte vulnerabilidades de seguranca por issues publicas do
GitHub.

Use o GitHub Private Vulnerability Reporting quando estiver disponivel:

https://github.com/lucas081297/fastlate/security/advisories/new

Voce tambem pode consultar a pagina de seguranca do repositorio aqui:

https://github.com/lucas081297/fastlate/security

Quando possivel, inclua:

- Uma descricao clara da vulnerabilidade.
- Passos para reproduzir o problema.
- A versao, commit ou configuracao afetada.
- O impacto potencial identificado.
- Sugestoes de correcao ou mitigacao, se houver.

## Divulgacao e Prazo de Resposta

Vamos tentar confirmar o recebimento de relatos de vulnerabilidade em ate 7
dias.

Depois da confirmacao, vamos investigar o relato, coordenar a correcao
necessaria e publicar uma atualizacao de seguranca assim que for pratico. Para
vulnerabilidades confirmadas, o objetivo e fornecer uma correcao ou mitigacao em
ate 90 dias sempre que possivel.

A divulgacao publica deve aguardar ate que uma correcao ou mitigacao esteja
disponivel, exceto em caso de exploracao ativa ou outro motivo claro de
seguranca publica para divulgar antes.

O credito sera dado a pessoa que reportou quando for apropriado e solicitado.
